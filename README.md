<h2>Идея проекта</h2>
Проект — единая платформа/модуль для статического и динамического тестирования диалоговых систем, объединяющая формальные проверки с помощью правил из базы знаний и с помощью LLM, также с использованием правил.
<h2>Определение и описание проблем потенциальных клиентов/заказчиков продукта</h2>
<p>Ручное тестирование дорого и не покрывает множества реальных запросов, а после релиза нередко всплывают ошибки — их замечают уже пользователи. 
Единая платформа/модуль предлагает проверку на уровне структуры диалогов, разговорного интерфейса, плюс метрики для их оценки и автоматического прогона тестов с генерацией запросов через LLM. Это помогает обнаружить пробелы в сценарии, ошибок по обработки естественного языка, снизить нагрузку на тестирование и уменьшить риск потери клиентов.</p>
<h2>Целевая аудитория проекта и предположения как проект будет решать проблемы каждого сегмента целевой аудитории</h2>
<p>Целевая аудитория: от разработчиков и интеграторов, которым нужны автоматические регрессионные прогоны и подробный анализ ошибок по структуре сценария или разговорного интерфейса, до менеджеров и QA-команд, ожидающих объективных метрик, рекомендаций для принятия решений. То есть аудитория у которых имеется готовый разработанный чат-бот, виртуальный ассистент, AI-агент.
Как проект будет решать проблемы: в статическом блоке анализируются дерево состояний (state), интенты, сущности, шаблоны ответов. В итоге получится список рекомендаций по исправлению ошибок структуры состояний, обработки естественного языка и список полученных значений по определенным метрикам оценки бота. В динамическом тестировании LLM генерируют обычные, вариативные запросы; система прогоняет их через диалоговую систему, сохраняет ответы. Далее ответы, которые дала диалоговая система проверяются на корректность, точность и правдоподобность.</p>
<h2>Цель проекта</h2>
<p>Цель — автоматизировать тестирование диалоговых систем для снижения затрат (денег, времени) на данное направление.
Задачи: определение логических ошибок в работе диалоговой системы до запуска; предоставление рекомендаций по структуре сценария и разговорному интерфейсу (сущности, намерения); оценка ответов диалоговой системы (корректность, точность и правдоподобность).</p>
